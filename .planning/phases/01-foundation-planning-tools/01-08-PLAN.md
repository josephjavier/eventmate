## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anonymous guest → public Route Handlers | No auth; token + minimal exposure are the only controls |
| guest submission → rsvp_responses | Must verify guest belongs to the token's event |
| client → guest Server Actions | Event-scoped via RLS + DAL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-12 | Information Disclosure | RSVP guest enumeration | mitigate | Search requires >= 2 chars, `.limit(5)`, returns only id/full_name/nickname/has_plus_one — no contact info |
| T-1-13 | Tampering | RSVP for a guest not on this event | mitigate | respond route validates `guest_id` belongs to the token's event_id before any write |
| T-1-14 | Tampering | RSVP after deadline | mitigate | respond route + page both gate on `isRsvpDeadlinePassed` (D-06) |
| T-1-06 | Elevation of Privilege | Service-role usage | mitigate | Service client used ONLY in the two public RSVP handlers + page token lookup; queries scoped to the token's event; no broad selects |
| T-1-01 | Information Disclosure | Cross-event guest writes | mitigate | Authenticated guest actions call `verifySession()`; guests/rsvp_responses RLS gated on `user_can_access_event` |
</threat_model>

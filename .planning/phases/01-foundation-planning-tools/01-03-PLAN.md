## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser cookie → server | JWT in HttpOnly cookie must be validated, not trusted as-is |
| Server Action / Route Handler → DB | DAL re-verifies session independent of middleware |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-05 | Spoofing | Forged session cookie | mitigate | `verifySession()` and middleware both use `supabase.auth.getUser()` which validates the JWT against the Supabase Auth server; `getSession()` is forbidden (acceptance criterion greps it out) |
| T-1-06 | Elevation of Privilege | Service-role key exposure | mitigate | `service.ts` uses `SUPABASE_SERVICE_ROLE_KEY` (no NEXT_PUBLIC_); file comment restricts usage to three public Route Handlers |
| T-1-07 | Tampering | Currency rounding | mitigate | Centralized `phpToCentavos`/`centavosToPhp` are the single rounding point; tests assert exact integer behavior |
</threat_model>

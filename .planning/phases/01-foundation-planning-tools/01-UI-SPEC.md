## Spacing Scale

All values are multiples of 4. This maps to Tailwind spacing utilities (`p-1` = 4px, `p-2` = 8px, etc.).

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon gaps, inline badge padding |
| sm | 8px | `gap-2` / `p-2` | Compact element spacing, icon-to-label gaps |
| md | 16px | `gap-4` / `p-4` | Default element spacing, form field vertical gaps |
| lg | 24px | `gap-6` / `p-6` | Card padding, section internal padding |
| xl | 32px | `gap-8` / `p-8` | Dashboard widget gaps, layout column gaps |
| 2xl | 48px | `gap-12` / `py-12` | Major section breaks, page header bottom margin |
| 3xl | 64px | `gap-16` / `py-16` | Page-level vertical padding |

**Exceptions:**

| Context | Value | Reason |
|---------|-------|--------|
| Touch targets (RSVP public page) | 44px minimum height | Public page accessed on mobile phones; WCAG 2.5.5 |
| File drop zone | 96px minimum height | Gives a clear drag target; matches react-dropzone convention |

## Typography

**Font family:** Plus Jakarta Sans (humanist sans-serif; warm, slightly rounded; legible at 14px–28px; loaded via `next/font/google` at weights 400 and 600 only).

```css
/* app/globals.css — Tailwind v4 CSS-first */
@import url('https://fonts.googleapis.com/css2?family=Plus+Jakarta+Sans:wght@400;600&display=swap');

:root {
  --font-sans: 'Plus Jakarta Sans', ui-sans-serif, system-ui, sans-serif;
}
```

| Role | Size | Weight | Line Height | Tailwind Class | Usage |
|------|------|--------|-------------|----------------|-------|
| Body | 16px | 400 | 1.5 | `text-base font-normal` | Paragraphs, table cells, form helper text, descriptions |
| Label | 14px | 600 | 1.4 | `text-sm font-semibold` | Form labels, table column headers, badge text, nav items |
| Heading | 20px | 600 | 1.2 | `text-xl font-semibold` | Section headings, card titles, dialog headings |
| Display | 28px | 600 | 1.1 | `text-3xl font-semibold` | Page-level titles (Dashboard, Checklist), dashboard countdown number |

> **Exactly 2 weights throughout the entire UI:** 400 (regular) and 600 (semibold).
> Do not use 500 (medium) or 700 (bold) anywhere in Phase 1.

## Color

shadcn base color: **stone**. These map directly to shadcn CSS variables in `app/globals.css`.
All color values below are the light-mode values (dark mode is not in scope for Phase 1).

| Role | Hex | shadcn Variable | Usage |
|------|-----|-----------------|-------|
| Dominant (60%) | `#FAFAF9` | `--background` | Page backgrounds, app shell, main content area |
| Secondary (30%) | `#F4EFE8` | `--card` / `--muted` | Cards, sidebar, nav background, table row alternates, input backgrounds |
| Accent (10%) | `#BE3C5E` | `--primary` | See reserved list below |
| Destructive | `#DC2626` | `--destructive` | Delete actions, over-budget alerts, error states only |

**Accent (`#BE3C5E`) is reserved exclusively for:**
1. Primary action buttons: "Set Up Your Wedding", "Create Your Event", "Confirm RSVP", "Start Planning"
2. Checklist progress bar fill
3. Budget progress bar fill (when category is under its allocation)
4. "Booked" and "Done" status badge backgrounds
5. Active navigation item indicator (left border or background)
6. Checkbox checked state (via shadcn `--primary`)

**Accent is NOT used for:**
- Secondary or ghost buttons
- Hover states on non-primary elements
- Body text or links (links use `--foreground` with underline)
- Informational or neutral states

> **Contrast check:** `#BE3C5E` on `#FAFAF9` = approximately 5.8:1 contrast ratio.
> Passes WCAG AA (4.5:1) for normal text. Passes WCAG AA for large text. ✓

## Page-Level Layout Contracts

### Auth Pages (`(auth)/login`, `signup`, `forgot-password`, `update-password`)
- Centered single card: max-width 400px, `mx-auto`, vertically centered
- Card padding: lg (24px)
- App logo above card, centered
- No navigation
- Cross-link below form: "Don't have an account? Sign up" / "Already have an account? Sign in"

### Onboarding Wizard (`/onboarding`)
- Centered single column: max-width 560px, `mx-auto`
- Step indicator at top: "Step N of N" — Label size (14px/600)
- No app navigation (standalone context)
- Back + Continue/Finish buttons in a row at bottom of form

### Client Dashboard — With Event (`(client)/dashboard`)
- Top bar: full-width, app logo left + user avatar/menu right
- Sidebar: 240px fixed left (desktop); deferred for mobile
- Main content: 2-column grid on `lg` screens; 1-column on smaller
- Widget order: 1) Countdown (full-width), 2) Checklist %, 3) Budget summary, 4) RSVP summary

### Countdown Widget
- Display: "{N} days until your wedding"
- Large number: 28px / 600 weight / Accent color (`#BE3C5E`) — Display size from Typography table
- Label below: "days until your wedding" — Body size (16px / 400)
- Must be `'use client'` component (hydration safety — RESEARCH.md Pitfall 6)
- Pass `eventDate: Date` as prop from Server Component parent

### Checklist Page (`(client)/event/checklist`)
- Categories as collapsible section groups (shadcn Collapsible or Card-based accordion)
- Each category shows: name, item count, completion badge
- Each item row: checkbox or status badge + item name + inline action menu
- "Add Item" button pinned below each category group

### Budget Page (`(client)/event/budget`)
- Summary bar at top: 4 stat cards in a row (Total Budget / Committed / Paid / Remaining)
- GCash-inspired layout: each stat is a large number with a label below — no chart required for MVP
- Category list below summary: each row shows category name + allocated ₱ + spent ₱ + thin progress bar
- Over-budget badge inline in the category row (destructive color)

### Guest List Page (`(client)/event/guests`)
- RSVP summary bar: 4 count badges (Invited / Going / Not Going / Pending) — full width
- "Generate RSVP Link" + "Add Guest" buttons in page header
- TanStack Table below: columns = Full Name, Nickname, +1, RSVP Status, Response Date, Actions
- Sortable by: Full Name, RSVP Status, Response Date

### Files Page (`(client)/event/files`)
- react-dropzone drop zone at top: full width, 96px min-height, dashed border
- File grid below: 3-column on desktop, 1-column on mobile
- Each file card: file type icon + filename (truncated) + tag + upload date + download/delete actions

### RSVP Public Page (`/rsvp/[token]`) — Open State
- No app navigation, no authentication UI
- Centered single column: max-width 480px, `mx-auto`
- App logo at top (minimal — not full nav)
- Search input → results list → RSVP form (sequential reveal)
- All interactive elements: 44px minimum height

### RSVP Public Page — Closed State (D-09)
- Same container as open state
- Closed message card at top
- Event details section: locations list with role labels
- Motif section: color motif text + attire reference photo(s) (max 2 images, static)

## Interaction Contracts

### Forms (all pages)
- State management: react-hook-form + Zod v4 + shadcn Form components
- Field error display: inline below the field, on blur or on submit attempt
- Error text: Label size (14px) in destructive color (`#DC2626`)
- Submit button: disabled + shows spinner while submitting
- Success feedback: sonner toast for all create/update/delete mutations
- Toast duration: 4 seconds for success; 6 seconds for errors

### Currency Inputs (Budget page)
- User types numeric peso amount: "50000" or "50,000"
- Input `type="text"` with `inputMode="numeric"` (numeric keyboard on mobile)
- Placeholder: "e.g., ₱ 50,000"
- Display formatted on blur: `Intl.NumberFormat('en-PH', { style: 'currency', currency: 'PHP', minimumFractionDigits: 0 })`
- Storage: `phpToCentavos(input)` → INTEGER centavos (CLAUDE.md locked — never float)
- Never display decimal places for whole peso amounts

### Checklist Status Flow
- **Supplier task:** Pending → Inquired → Booked (Phase 4 auto-updates; schema supports FK now)
- **Personal task:** Pending → Done (single checkbox toggle)
- Status badge variants:
  - Pending: `variant="outline"` (border only, no fill)
  - Inquired: `variant="secondary"` (muted background)
  - Booked / Done: Accent fill (`#BE3C5E` background, white text)

### File Upload Flow
- Drop zone: dashed border in muted color; accent border color on drag-over
- Accepted types: PDF, JPG, PNG — reject others immediately client-side (react-dropzone `accept` prop)
- Size limit: 10 MB — reject before upload attempt (react-dropzone `maxSize`)
- During upload: show filename + spinner in a list row
- On success: file card appears in grid, sonner success toast
- On failure: sonner error toast with specific message from the global error copy table

### RSVP Name Search
- Trigger: minimum 2 characters typed
- Debounce: 300ms before API call
- Results format: "Maria Santos (Tita)" when nickname exists — per D-05 disambiguation
- Result limit: 5 per search (server enforced)
- No result: show "Name not found" inline message (see Copywriting Contract)

### Budget Over-Limit Alert
- Trigger: `SUM(expenses.total_amount) > budget_categories.allocated_amount` (client-side comparison)
- Display: destructive Badge "Over budget by ₱X,XXX" inline in the category row
- No modal, no blocking — purely informational
- Badge disappears when the total falls back under allocation

### Countdown Timer
- Calculation: `differenceInDays(eventDate, new Date())` from date-fns
- Component: `'use client'` — never render in a Server Component (RESEARCH.md Pitfall 6)
- If 0 days: "Today is your wedding day!"
- If negative (event passed): "X days since your wedding"

### Accessibility — Icon-Only Buttons
- All icon-only action buttons (e.g., delete, edit, download in table rows and file cards) must include an `aria-label` matching the action text (e.g., `aria-label="Delete item"`, `aria-label="Download file"`).
